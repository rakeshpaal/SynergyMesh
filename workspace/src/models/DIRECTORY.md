# models

## 目錄職責

此目錄為 MachineNativeOps 的**資料模型層**，定義系統的資料結構，包含資料庫模型、資料傳輸物件 (DTO) 和 Schema 驗證。

## 子目錄說明

| 子目錄 | 職責 |
|--------|------|
| `database/` | 資料庫模型 - ORM 實體定義 |
| `dto/` | 資料傳輸物件 - API 請求/回應結構 |
| `schemas/` | Schema 驗證 - 資料驗證規則 |

## 模型類型說明

### Database Models
- **用途**：定義資料庫表結構
- **特性**：ORM 映射、關聯定義、索引設定

### DTO (Data Transfer Objects)
- **用途**：定義 API 層的資料結構
- **特性**：輸入驗證、輸出格式化、版本控制

### Schemas
- **用途**：定義資料驗證規則
- **特性**：型別檢查、約束驗證、預設值

## 目前狀態

此目錄為骨架結構，模型定義待補充。

## 設計原則

1. **分層清晰**：資料庫模型與 DTO 分離
2. **單一職責**：每個模型只負責一種實體
3. **不可變性**：DTO 設計為不可變物件
4. **驗證優先**：所有輸入都需經過 Schema 驗證

## 依賴規則

**可以依賴**：
- `src/shared/` - 共用類型定義
- 驗證庫（Zod, Pydantic 等）

**不可依賴**：
- `src/services/` - 模型層不應依賴服務層
- `src/core/` - 模型層不應依賴核心引擎
- `src/api/` - 模型層不應依賴 API 層

## 與其他目錄的關係

- **src/services/**：服務層使用模型進行資料操作
- **src/api/**：API 層使用 DTO 定義請求/回應
- **src/shared/types/**：共用類型可被模型引用
