# 文檔治理 - 自檢清單

## 設計前規劃階段

- [ ] 文檔範圍與目標讀者是否已明確？
- [ ] 預期更新頻率與維護負責人是否已定義？
- [ ] 文檔與代碼同步策略是否已規劃？
- [ ] 是否需要多語言支援，若需要哪些語言？

## 設計中撰寫階段

- [ ] 文檔是否遵循組織風格指南與格式標準？
- [ ] 示例與代碼片段是否清晰且可運行？
- [ ] 是否添加了機器可讀的元數據（YAML 前置、標籤等）？
- [ ] 是否包含指向相關文檔與外部資源的交叉參考？

## 實現前驗證

- [ ] 文檔內容準確性是否已驗證（與代碼同步）？
- [ ] 是否包含故障排除、常見問題與限制部分？
- [ ] 是否包含清晰的先決條件與依賴說明？
- [ ] 文檔格式與 Markdown 語法是否通過 lint 檢查？

## 上線前最終檢查

- [ ] 文檔是否已添加到主索引與知識庫？
- [ ] 拼寫、語法與格式是否已驗證？
- [ ] 文檔版本與發佈日期是否已標記？
- [ ] 是否已通知相關受眾（團隊、用戶、外部合作者）？
- [ ] 文檔維護與更新週期是否已排定？
