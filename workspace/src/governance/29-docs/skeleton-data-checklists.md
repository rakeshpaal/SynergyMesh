# 資料治理 - 自檢清單

## 設計前分析階段

- [ ] 需要哪些資料及其來源是什麼？
- [ ] 如何按敏感度與用途分類？
- [ ] 適用的隱私與合規要求是什麼（GDPR、CCPA 等）？
- [ ] 資料流向涉及哪些司法管轄區（跨境轉移）？

## 設計中架構階段

- [ ] 資料模式（Schema）與欄位定義是否已明確？
- [ ] 資料流向與轉換路徑是否已追蹤且文件化？
- [ ] 敏感資料是否已標記（PII、財務、健康等）？
- [ ] 資料隔離與存取控制設計是否充分？

## 實現前準備

- [ ] 資料驗證與品質檢查是否已實現？
- [ ] 隱私保護措施是否已啟用（加密、匿名化等）？
- [ ] 審計日誌與追蹤機制是否已配置？
- [ ] 資料保留與銷毀政策是否已自動化？

## 上線前合規驗證

- [ ] 隱私影響評估（PIA）是否已通過？
- [ ] 合規性驗證（GDPR、CCPA 等）是否已完成？
- [ ] 用戶隱私通知與同意是否已準備？
- [ ] 資料主體權利實現機制（訪問、刪除、轉移）是否已測試？
- [ ] 法律與隱私團隊是否已批准上線？
