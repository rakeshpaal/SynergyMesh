# API Governance Guardrails

## 硬性邊界

1. **禁止無文件的 API**
   - 所有公開 API 必須有 OpenAPI/gRPC proto 定義
   - 定義必須在代碼審查時同時更新

2. **禁止無版本的破壞性變更**
   - 移除欄位、改變參數類型等破壞性變更必須開新版本
   - 舊版本必須保留至少 6 個月

3. **禁止未文件化的棄用**
   - API 棄用必須提前 90 天通知
   - 必須提供遷移指南和替代方案

4. **禁止不相容的預設值**
   - 新增欄位必須有合理的預設值
   - 不得改變既有欄位的預設值
