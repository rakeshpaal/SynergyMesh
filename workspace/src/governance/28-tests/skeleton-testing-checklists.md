# 測試治理 - 自檢清單

## 設計前風險分析

- [ ] 業務流程與使用者場景是否已分析？
- [ ] 高風險與關鍵路徑是否已識別？
- [ ] 測試預算與時間限制是否已分配？
- [ ] 測試利害相關者與審批流程是否已明確？

## 設計中策略規劃

- [ ] 測試場景與測試用例是否完整（Happy Path、邊界、異常）？
- [ ] 覆蓋率目標是否現實且與風險等級相符？
- [ ] 測試自動化投資是否可行與值得？
- [ ] 測試金字塔結構是否適當（單元>整合>E2E）？

## 實現前準備

- [ ] 測試框架與工具是否已選定與配置？
- [ ] Mock 與 Stub 策略是否已定義？
- [ ] 測試數據與測試環境是否已準備？
- [ ] 品質閘條件與通過/失敗標準是否已明確？

## 上線前最終檢查

- [ ] 所有測試是否通過（包括邊界情況與負面測試）？
- [ ] 覆蓋率是否達到目標（>85% 代碼覆蓋、>90% 關鍵路徑）？
- [ ] 效能與安全測試是否通過？
- [ ] 缺陷是否已分類與優先級化，高風險缺陷是否已修復？
- [ ] 測試策略與執行結果是否已文件化供未來參考？
