# 測試治理 - 不可越界

## 禁止操作

- ❌ **跳過品質門檛直接部署** — 所有代碼變更必須通過最低覆蓋率（85%）與關鍵測試
- ❌ **降低覆蓋率要求以加快發布** — 覆蓋率下降需明確的風險評估與批准
- ❌ **測試只覆蓋 Happy Path 而忽視邊界情況** — 必須包含異常、邊界與故障路徑測試
- ❌ **禁用或跳過自動化品質閘** — 品質閘必須強制執行；例外需技術與管理雙重批准
- ❌ **依賴人工測試而不進行自動化** — 關鍵路徑必須自動化；人工測試僅用於補充

## 人類介入觸發條件

- 🔴 **覆蓋率下降超過 5%** — 需根本原因分析與改進計畫
- 🔴 **高風險代碼（如支付、安全、基礎設施）低覆蓋率** — 需強制提升覆蓋率
- 🔴 **品質閘失敗需忽略** — 需技術主管與 QA 負責人簽核
- 🔴 **新的測試標準或工具** — 需測試委員會評估與實施計畫
- 🔴 **重大測試失敗或 bug 逃逸** — 需事後檢討與測試策略改進
