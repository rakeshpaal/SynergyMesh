# archive

## 目錄職責

此目錄用於存放**已棄用或歸檔**的歷史資產。這些內容已不再活躍維護，但保留作為參考或未來可能的遷移來源。

## 子目錄說明

| 子目錄 | 狀態 | 說明 |
|--------|------|------|
| `experiments/` | 空 | 預留給實驗性代碼的歸檔區，目前僅含 `.gitkeep` |
| `infrastructure/` | 歸檔 | 舊版基礎設施配置（Kubernetes manifests、監控、漂移檢測）|
| `legacy/` | 歸檔 | 舊版系統代碼：v1-python-drones、v2-multi-islands |
| `unmanned-engineer-ceo/` | 歸檔 | 「無人工程師 CEO」能力模型與知識庫系統 |

## 各子目錄詳情

### experiments/
- **內容**：空目錄，僅含 `.gitkeep` 保持目錄結構
- **用途**：預留給未來實驗性功能的歸檔

### infrastructure/
- **內容**：Kubernetes 編排、Prometheus/Grafana 監控、漂移檢測腳本
- **來源**：早期 SynergyMesh 基礎設施配置
- **注意**：現行配置已遷移至 `deploy/` 和 `ops/`

### legacy/
- **v1-python-drones**：第一版 Python 無人機自動化系統
- **v2-multi-islands**：第二版多島嶼架構系統
- **用途**：保留歷史實作供參考，已不建議直接使用

### unmanned-engineer-ceo/
- **內容**：虛擬工程師總執行長能力模型，包含 00-foundation 至 99-meta 共 14 個層級
- **範圍**：技術基礎、進階工程、企業能力、智能系統、評估矩陣、模擬演練等
- **注意**：部分檔案存在 git merge conflict 標記，需人工處理

## 變更影響範圍

- 此目錄內容為**唯讀歸檔**，修改不影響生產系統
- 若需參考舊版實作，應複製到工作目錄後再修改
- 刪除子目錄前應確認無其他模組引用

## 維護原則

1. **不主動維護**：除非有明確遷移需求，否則不更新此目錄內容
2. **保持完整**：保留完整歷史記錄，不隨意刪除
3. **標記狀態**：若有內容被重新啟用，應移出此目錄並更新文檔
