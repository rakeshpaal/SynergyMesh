# 知識庫骨架 / Knowledge Base Skeleton

## 目的

組織、索引與管理系統內所有可用的知識資源，包括文檔、API 契約、架構圖、最佳實踐、設計決策、決策紀錄（ADR）與常見問題。AI 在查詢系統架構、尋求最佳實踐或尋找決策根據時查詢此骨架。

## AI 使用情境

- 查詢系統架構與設計決策依據
- 搜尋 API 契約與實現範例
- 查找特定問題域的最佳實踐指南
- 評估新設計是否與現有架構一致
- 追蹤決策演變與棄用理由
- 生成知識圖譜與架構可視化

## 責任

- ✅ 知識組織與索引維護
- ✅ 知識檢索與搜尋引擎
- ✅ 知識版本管理與更新追蹤
- ✅ 知識完整性檢查（孤立文檔、斷鏈）
- ✅ 知識圖譜構建與維護
- ✅ 決策紀錄（ADR）管理
- ✅ 多維度分類與標籤體系

## 非目標

- ❌ 知識生成與撰寫（由各模組團隊負責）
- ❌ 文檔撰寫標準與流程（由 Docs Governance 骨架負責）
- ❌ 知識庫基礎設施與搜尋引擎實現（由 Infrastructure 骨架負責）
- ❌ 內容管理系統（CMS）開發（由工具團隊負責）
