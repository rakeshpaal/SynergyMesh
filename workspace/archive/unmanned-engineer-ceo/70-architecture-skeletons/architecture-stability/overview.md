# Architecture Stability Skeleton

## 目的

本 skeleton 定義系統的分層架構、依賴規則與架構不變條件，包括：

- **分層模型**：core → platform → services → agents → applications
- **依賴規則**：單向依賴、禁止循環、同層隔離
- **架構不變條件**：資料所有權、非同步優先、零信任原則
- **架構驗證**：自動化檢查工具、違規檢測

## AI 在進行以下操作時必須使用本 skeleton

- 設計新的服務模組或代理
- 評估架構變更的依賴影響
- 新增跨層級或跨服務的通訊
- 規劃系統分解與模組化策略

## 核心問題

- 這個模組應該在哪個層級？
- 它可以依賴什麼？
- 這是否違反了架構不變條件？
- 如何確保向前相容？

## 非目標

- 不負責安全考量（交由 security-observability）
- 不負責資料模型設計（交由 data-governance）
- 不負責部署拓撲（交由基礎設施層）
