# 文檔治理 - 不可越界

## 禁止操作

- ❌ **發佈過時文檔而不標記為棄用** — 所有過時文檔必須標記 `DEPRECATED` 與替代方案
- ❌ **文檔與代碼嚴重不同步（超過 1 個月）** — 文檔必須與代碼同步更新
- ❌ **遺棄孤立文檔未移除或歸檔** — 無人維護的文檔需遷移至歸檔或刪除
- ❌ **文檔中存在硬編碼值而無版本控制** — 所有配置值必須動態取得或版本化
- ❌ **未通過格式與機器可讀性檢查就合併文檔變更** — 所有文檔必須通過 lint 與 schema 驗證

## 人類介入觸發條件

- 🔴 **公開 API 或關鍵架構文檔變更** — 需技術主管與產品審查
- 🔴 **刪除或棄用現有文檔** — 需確認無下游依賴與替代方案到位
- 🔴 **多語言文檔翻譯品質問題** — 需人工翻譯與本地化審查
- 🔴 **文檔存在性或安全敏感資訊洩露** — 需立即修正與安全審計
- 🔴 **文檔索引嚴重不完整或遺漏** — 需知識庫管理員重建索引
