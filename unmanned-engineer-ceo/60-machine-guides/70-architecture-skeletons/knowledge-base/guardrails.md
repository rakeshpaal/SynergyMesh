# 知識庫 - 不可越界

## 禁止操作

- ❌ **手動編輯機器生成的知識而不保持同步**
  — 機器生成的知識必須通過自動化流程更新
- ❌ **存儲過時或廢棄的知識不更新或移除** — 廢棄知識需標記 `DEPRECATED`
  與替代方案
- ❌ **知識版本不一致跨多系統** — 知識庫必須是單一真實來源（SSOT）
- ❌ **未授權或未認證存取敏感知識** — 敏感知識（如架構安全隱患）需存取控制
- ❌ **知識圖譜斷鏈或遺漏關鍵依賴** — 必須維持完整的知識關聯與追溯能力

## 人類介入觸發條件

- 🔴 **架構決策或重大設計決定** — 需產出 ADR 並納入知識庫
- 🔴 **知識庫重大結構重組** — 需資訊架構或知識主管審批
- 🔴 **處理機器生成知識與人工知識的衝突** — 需人工審查與決策
- 🔴 **刪除或棄用知識** — 需確認無下游依賴與替代方案
- 🔴 **知識完整性檢查失敗** — 需診斷並修復斷鏈或遺漏的知識
