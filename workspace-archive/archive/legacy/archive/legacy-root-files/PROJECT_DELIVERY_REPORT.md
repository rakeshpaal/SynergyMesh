## ✨ 核心功能展示

### 1. 終端機命令偵錯

```bash
# 啟動偵錯
$ mno debug start --config "Python: Debug Current File"
✅ Created debug session: session_1
🚀 Debug session started

# 設定斷點
$ mno debug breakpoint src/main.py 10
✅ Breakpoint 1 set at src/main.py:10

# 執行控制
$ mno debug continue
▶️  Continuing execution...

$ mno debug next
⏭️  Stepped over
📍 src/main.py:11

# 查看變數
$ mno debug variables
📊 Local Variables:
  • x = 42 (int)
  • y = 10 (int)
  • result = 52 (int)
```

### 2. 互動式 REPL

```
$ mno debug repl
🔧 MachineNativeOps Debug REPL

(mno-debug) start Python: Debug Current File
✅ Debug session started

(mno-debug) break src/main.py 10
✅ Breakpoint 1 set

(mno-debug) continue
▶️  Program running...

(mno-debug) vars
📊 Variables: x=42, y=10

(mno-debug) eval x + y
💡 x + y = 52 (int)
```

### 3. 聊天式偵錯

```
User: 啟動偵錯

AI: ✅ 已啟動偵錯會話：Python: Debug Current File

    您可以：
    - 設定斷點
    - 開始執行
    - 檢視變數

User: 在第 10 行設定斷點

AI: ✅ 已在 src/main.py 的第 10 行設定斷點。
    
    要繼續執行嗎？

User: 為什麼程式在第 10 行崩潰？

AI: 🔍 錯誤分析：

    **錯誤類型**：ZeroDivisionError
    **說明**：除以零錯誤：嘗試將數字除以零。
    
    **可能原因**：
      • 變數值為 0
      • 計算結果為 0
      • 未初始化的變數
    
    **建議修復方法**：
      • 在除法前檢查除數是否為 0
      • 使用 try-except 捕獲異常
      • 確保變數正確初始化
```

## 🚀 部署指南

### 快速安裝

```bash
# 1. 克隆儲存庫
git clone https://github.com/MachineNativeOps/MachineNativeOps.git
cd MachineNativeOps

# 2. 執行安裝腳本
./scripts/setup-debug-system.sh

# 3. 開始使用
python -m src.core.run-debug.cli repl
```

### 手動安裝

```bash
# 1. 安裝依賴
pip install -r requirements-debug.txt

# 2. 建立配置
mkdir -p .vscode
cp examples/debug-examples/launch.json .vscode/

# 3. 執行示範
python examples/debug-examples/demo.py
```
