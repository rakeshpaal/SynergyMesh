# 快速參考 / Quick Reference

## 🎯 立即開始 / Get Started Now

### 第一次使用？

```
1. 開啟此目錄
2. 讀取 README.md 了解系統
3. 查看 skeletons-index.yaml 找到相關骨架
4. 進入骨架目錄
5. 按順序讀取 5 個檔案
```

| 📊 資料治理 | 資料模式、隱私合規 | 設計資料架構 |
| 🧪 測試治理 | 測試策略、覆蓋率 | 規劃測試 |
| ⚡ 性能與可靠 | SLA、容量、故障恢復 | 優化性能 |
| 💰 成本管理 | 成本監控、預算規劃 | 成本優化 |

| 🧠 知識庫 | 知識組織、查詢 | 查詢知識 |

### 每個骨架的 5 個檔案

```
每個骨架/
├── overview.md              ← 「我是做什麼的」

```

## 💡 常見問題

### Q：我應該按什麼順序讀骨架？

A：根據任務需求。通常先讀 Architecture Stability，再讀相關的特定骨架。

### Q：我需要讀所有 11 個骨架嗎？

A：不需要。只讀與你的任務相關的骨架。但要了解它們之間的依賴關係。

### Q：如果我違反了 guardrails 怎麼辦？

A：1. 停止操作 2. 記錄問題 3. 尋求架構師評審 4. 文檔化異常決策

### Q：我在哪裡可以找到具體的代碼位置？

A：查看相應骨架的
`runtime-mapping.yaml`。它列出了所有相關的代碼、配置、文檔位置
